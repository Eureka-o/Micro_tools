隐私优先本地工具箱。

有些工具应该默认本地运行。Token、payload 样本、密码、文件片段和加密测试数据，正是不应该为了“解码一下”或“算个哈希”就发送到陌生服务器的内容。这个合集收纳了 Micro Tools Matrix 中最常与敏感文本或文件一起使用的工具。

你可以用 JWT 解码器查看 Header 和 Payload claims，用 Base64 工具转换文本或文件，用哈希工具校验摘要，用密码生成器本地生成候选密码，用二维码工作台生成可扫描图片而不依赖远程图片服务。涉及真正安全决策时，仍应使用后端签名验证和经过审计的密码学库；这些页面负责本地工作流辅助。

这里的隐私优先规则很直接：常规工具使用不走后端上传，不使用外部运行库 CDN，也不强制加载广告或统计脚本。未来可能需要赞助或统计时，当前也只保留本地占位或事件接口，而不是网络依赖。

这对复访尤其重要：开发者可能在一次联调中先解码 JWT，再检查 JSON payload，随后计算回调内容哈希，最后为本地测试设备生成二维码。页面之间建立了明确连接，让这些步骤容易发现，同时敏感输入仍然停留在设备上。

常见问题

隐私优先等于匿名吗？

不是。它表示工具设计为本地处理、不上传输入；浏览器历史、截图和你自己的分享行为仍需注意。

JWT 解码器能用于生产授权判断吗？

不能。它只解码，生产系统必须在服务端验证签名和 claims。

赞助占位是正在运行的广告吗？

不是。它只是静态预留区域，不加载广告脚本。
